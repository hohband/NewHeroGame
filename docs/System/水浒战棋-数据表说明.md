# 《水浒战棋》数据表 Schema 说明

> 版本 V1.0 ｜ 2026年7月 ｜ 配套CSV：武将表（24行）、地形表（8行）、技能表（26行）
> CSV 均为 UTF-8 with BOM 编码，Excel 与 Godot 均可直接读取。**CSV是唯一数据源**，禁止在代码里硬编码数值。

---

## 一、武将表（水浒战棋-数据表-武将.csv）

| 字段 | 类型 | 说明 |
|---|---|---|
| unit_id | string | 全局唯一ID，蛇形命名（lin_chong），资源/代码引用一律用ID不用中文名 |
| name / nickname / star | string | 姓名 / 绰号 / 星宿，显示与文案用 |
| quality | enum | `orange` / `purple` / `blue` / `green` |
| class | enum | `vanguard`先锋 / `infantry`步军 / `cavalry`马军 / `archer`神射 / `strategist`谋士 / `healer`医者 / `support`辅助 |
| hp / atk / def / mgc / spd | int | Lv.1基础值：生命/攻击/防御/谋略/速度 |
| crit / dodge / block | int | 暴击率/闪避率/格挡率，单位% |
| move | int | 基础移动力（格） |
| range_min / range_max | int | 普攻范围；近战为1 |
| weapon | string | 专属武器名，关联武器范围模板表（weapons.csv，见第六节） |
| skill_signature | string | 招牌技能ID，关联技能表 |
| bonds | string | 羁绊列表，格式 `目标unit_id或预留名\|羁绊名;…`；预留名指尚未实装的武将（如 xu_ning） |
| unlock | string | 获取方式（剧情/挑战/招募/初始） |

**数值基调**：蓝将为基准值，紫将约+8%，橙将约+15%，绿将约-8%；职业间按定位差异化（先锋高血防、谋士高谋略、戴宗速度全场最高95）。当前数值为原型期占位值，需在测试中调整。

## 二、地形表（水浒战棋-数据表-地形.csv）

| 字段 | 类型 | 说明 |
|---|---|---|
| terrain_id | string | 唯一ID |
| move_cost | int | 进入消耗；99表示不可通行 |
| dodge_mod / def_mod / atk_mod | int | 闪避/防御/攻击修正，单位% |
| range_mod | int | 射程修正（山地+1） |
| passable / destructible | 0/1 | 是否可通行 / 是否可破坏 |
| hp | int | 可破坏物耐久（拒马300，约3次攻击） |
| special | string | 特殊效果描述（脚本钩子按terrain_id实现） |

## 三、技能表（水浒战棋-数据表-技能.csv）

| 字段 | 类型 | 说明 |
|---|---|---|
| skill_id | string | 唯一ID，`ult_` 绝技 / `act_` 主动 / 被动统一用 `pas_`（后续补充） |
| owner | string | 持有武将ID；`通用` 表示所有单位可用 |
| type | enum | `active` / `passive` / `ult` |
| trigger | string | 触发方式：主动技能为 `manual`；被动为事件名（on_attack / on_hit / turn_start 等） |
| range_shape | enum | `adjacent`相邻 / `line`直线 / `ring`周身 / `diamond`菱形 / `all`全图 / `self`自身 |
| range_min / range_max | int | 范围距离 |
| target | enum | `enemy` / `ally` / `self` |
| cooldown | int | 冷却回合数（主动技2—4） |
| rage_cost | int | 怒气消耗，绝技统一100 |
| effects | string | **原子效果序列**，分号分隔，按顺序执行 |
| desc | string | 技能文案（显示用） |

### 原子效果词表（effects字段的合法词汇）

| 类别 | 效果 |
|---|---|
| 伤害/治疗 | `phys_dmg(倍率)` `mgc_dmg(倍率)` `heal(倍率)` `bleed(回合)` `burn(回合)` `poison(回合)` |
| 位移 | `pull(格)` `pull_to_front` `push(格)` `swap_position` `teleport(格)` |
| 控制 | `stun(回合)` `sleep(回合)` `sleep_chance(概率,回合)` `paralyze(回合)` `bind(回合)` |
| 增减益 | `buff(属性,值,回合)` `debuff_mgc(值,回合)` `def_up(值,回合)` `dodge_up(值,回合)` `block_up(值,回合)` `armor_break(值,回合)` `move_mod(值,回合)` `random_buff(A\|B)` |
| 机制 | `steal_buff(数)` `dispel(数)` `summon(物件)` `aura(属性,值,半径)` `guard(回合)` `counter(回合)` `extra_action(数)` `av_mod(值)` `refresh_on_kill(1)` `execute_below(比例)` `sure_hit` `random_target(数)` `friendly_fire(概率)` `hit_rate(值)` `rage(值)` |
| 修正 | `bonus_vs_elite(倍率)` `bonus_vs_high_def(值)` `bonus_vs_cavalry(值)` `bonus_by_self_lost_hp(系数)` `target_rule(lowest_hp)` `target_rule(random)` `chance(概率)` |
| 连击 | 任何伤害效果后可跟 `xN` 表示N段（如 `phys_dmg(0.9)x4`） |

**新增技能 = 在表中加一行拼效果，程序零介入**（对应策划文档6.7效果原子化）。未收录的效果需先在 EffectSystem 中实现并更新本表。

### 被动触发语义（策划文档 4.3：每名武将 1 主动 + 2 被动 + 1 绝技）

- `type=passive` 的技能不手动施放，由战斗在结算点自动触发；`trigger` 列即触发事件：`on_attack`（单位攻击命中结算后）/ `on_hit`（单位被攻击命中后）/ `turn_start`（单位激活开始时）。主动技/绝技恒为 `manual`。
- **目标语义**：被动没有手动选目标。`target=self` 的效果作用于持有者自身（增益/治疗/回怒/驱散/光环类）；`target=enemy` 的效果作用于「涉事对方」——on_attack 指被攻击者，on_hit 指攻击者（伤害/减益/控制类）。一行被动只写一个目标域；turn_start 没有涉事对方，`target` 必须为 `self`。
- 书写约束：`owner` 必填持有武将 ID（每名武将恰好 2 行被动，validate 校验）；`cooldown`/`rage_cost` 恒为 0；`range_shape`/`range_min`/`range_max` 占位不用（统一 `self,0,0`）。
- **概率**：整串按概率触发用修饰词 `chance(概率)`（如 `chance(0.35);phys_dmg(0.25)`，未触发不产事件）；单个控制效果的概率沿用 `sleep_chance(概率,回合)` 这类自带判定词。判定均经 RollSource。
- 结算细节：被动效果串与主动技共用 EffectSystem 同一执行路径；被动造成的伤害不再触发反击或连锁被动（深度口径同 counter），也不产受击/击杀怒气（附带伤害不给对面攒绝技，决策日志 D41）；被命中目标已死亡时 on_hit/on_attack 的 enemy 域被动跳过。事件复用现有类型（damage/buff/status/heal 等），前置 `passive_trigger` 标记事件供表现层提示。

## 四、Godot 接入约定

1. 启动时由 `DataLoader`（AutoLoad）解析三张CSV → 生成 `UnitData` / `TerrainData` / `SkillData` 自定义Resource并缓存，全项目通过ID查询
2. 战斗内所有数值读取必须走Resource，禁止硬编码
3. `effects` 字段由 EffectSystem 按词表逐个解析执行；解析失败要报错并指出技能ID与效果名
4. 调参流程：策划改CSV → 重进游戏即生效，不改代码、不重导出

## 六、武器范围模板表（weapons.csv）

| 字段 | 类型 | 说明 |
|---|---|---|
| weapon | string | 武器名，与武将表/敌人表 weapon 列一一对应 |
| range_shape | enum | 普攻范围模板，词表同技能表：`line`直线 / `adjacent`相邻 / `diamond`菱形（其余 `ring`/`all`/`self` 普攻暂不用） |
| desc | string | 武器说明（显示用） |

**口径**（策划 6.5；决策日志 D9 的细化落地）：

- 数值范围沿用单位自身 `range_min/range_max`，本表只定形状、不重复数值
- `line` = 四方向直线 range_min..range_max 格（枪/矛类；只看格子在直线上，不做遮挡判定）；`adjacent`/`diamond` = 曼哈顿区间（近战 1 格时两者等价）
- 山地 `range_mod` 加成照常生效：射程上限延伸，形状不变
- 未登记的武器按 diamond 处理并告警；validate 校验：武将表/敌人表出现的每个 weapon 必须在本表有行

## 七、道具表（items.csv）

| 字段 | 类型 | 说明 |
|---|---|---|
| item_id | string | 唯一ID，蛇形命名（jinchuangyao） |
| name | string | 道具名（显示用） |
| range_shape | enum | 范围模板，词表同技能表：`adjacent` / `line` / `ring` / `diamond` / `all` / `self` |
| range_min / range_max | int | 范围距离 |
| target | enum | `enemy` / `ally` / `self` |
| uses_per_battle | int | 每局限用次数（≥1，validate 校验） |
| effects | string | **原子效果序列**，词表同技能表第三节（不加道具专属新效果） |
| desc | string | 道具文案（显示用） |

**口径**（策划 6.5 行动类型「道具：每局限用次数」）：

- 道具 = 效果序列 + 目标规则，结算借道 Targeting/EffectSystem（ItemData 投影为 SkillData），程序零介入：新道具 = 本表加一行
- 使用道具是一次「行动」，与攻击/技能同级（行动后不可再攻击/施技/再用道具）；移动（含分段）不受影响
- 次数按场记：开局按本表建立标准道具栏（全员通用），`uses_per_battle` 为每件道具本场总次数；meta/背包对接经 `BattleManager.set_item_stock()` 覆盖
- 目标为单目标指定：玩家在范围内点选 1 个合法目标生效（脚本化不指定时退回技能同口径 = 范围内全体）
- 道具不是攻击/技能：不触发攻击类被动、不产施放回怒（rage_on_skill）；伤害类道具的受击/击杀回怒走伤害统一路径
- `refresh_on_kill` / `extra_action` 为技能指令专属后处理，道具不生效（validate 拦截）；AI 不使用道具

## 八、数据校验规则（建议做成编辑器工具脚本）

- 所有 `skill_signature` 必须能在技能表中找到
- 所有 `bonds` 中的ID必须是已实装或登记过的预留名
- 武将表/敌人表的每个 `weapon` 必须在 weapons.csv 有登记行，且 `range_shape` 在枚举范围内
- `quality`、`class`、`type`、`trigger`、`range_shape`、`target` 必须在枚举范围内（trigger：manual / on_attack / on_hit / turn_start）
- **被动规则（4.3）**：武将表每名武将恰好 2 个 `type=passive` 技能（owner 匹配）；被动行 trigger 必须非 manual；target 限 self/enemy；trigger=turn_start 时 target 必须为 self；cooldown/rage_cost 为 0
- 同名ID不得重复；数值不得为负（move_cost 99 除外）
- battle_constants.csv 必须存在且包含全部必备 key（见第五节）
- **道具规则（6.5）**：items.csv 每行 range_shape/target 在枚举内；uses_per_battle ≥1；range_min ≤ range_max 且非负；effects 逐词可解析且全部落在已实现原子词表内；不得使用技能指令专属的 refresh_on_kill / extra_action
