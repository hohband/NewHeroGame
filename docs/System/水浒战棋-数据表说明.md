# 《水浒战棋》数据表 Schema 说明

> 版本 V1.0 ｜ 2026年7月 ｜ 配套CSV：武将表（24行）、地形表（8行）、技能表（26行）
> CSV 均为 UTF-8 with BOM 编码，Excel 与 Godot 均可直接读取。**CSV是唯一数据源**，禁止在代码里硬编码数值。

---

## 一、武将表（水浒战棋-数据表-武将.csv）

| 字段 | 类型 | 说明 |
|---|---|---|
| unit_id | string | 全局唯一ID，蛇形命名（lin_chong），资源/代码引用一律用ID不用中文名 |
| name / nickname / star | string | 姓名 / 绰号 / 星宿，显示与文案用 |
| quality | enum | `orange` / `purple` / `blue` / `green` |
| class | enum | `vanguard`先锋 / `infantry`步军 / `cavalry`马军 / `archer`神射 / `strategist`谋士 / `healer`医者 / `support`辅助 |
| hp / atk / def / mgc / spd | int | Lv.1基础值：生命/攻击/防御/谋略/速度 |
| crit / dodge / block | int | 暴击率/闪避率/格挡率，单位% |
| move | int | 基础移动力（格） |
| range_min / range_max | int | 普攻范围；近战为1 |
| weapon | string | 专属武器名（武器表预留，后续扩展） |
| skill_signature | string | 招牌技能ID，关联技能表 |
| bonds | string | 羁绊列表，格式 `目标unit_id或预留名\|羁绊名;…`；预留名指尚未实装的武将（如 xu_ning） |
| unlock | string | 获取方式（剧情/挑战/招募/初始） |

**数值基调**：蓝将为基准值，紫将约+8%，橙将约+15%，绿将约-8%；职业间按定位差异化（先锋高血防、谋士高谋略、戴宗速度全场最高95）。当前数值为原型期占位值，需在测试中调整。

## 二、地形表（水浒战棋-数据表-地形.csv）

| 字段 | 类型 | 说明 |
|---|---|---|
| terrain_id | string | 唯一ID |
| move_cost | int | 进入消耗；99表示不可通行 |
| dodge_mod / def_mod / atk_mod | int | 闪避/防御/攻击修正，单位% |
| range_mod | int | 射程修正（山地+1） |
| passable / destructible | 0/1 | 是否可通行 / 是否可破坏 |
| hp | int | 可破坏物耐久（拒马300，约3次攻击） |
| special | string | 特殊效果描述（脚本钩子按terrain_id实现） |

## 三、技能表（水浒战棋-数据表-技能.csv）

| 字段 | 类型 | 说明 |
|---|---|---|
| skill_id | string | 唯一ID，`ult_` 绝技 / `act_` 主动 / 被动统一用 `pas_`（后续补充） |
| owner | string | 持有武将ID；`通用` 表示所有单位可用 |
| type | enum | `active` / `passive` / `ult` |
| trigger | string | 触发方式：主动技能为 `manual`；被动为事件名（on_attack / on_hit / turn_start 等） |
| range_shape | enum | `adjacent`相邻 / `line`直线 / `ring`周身 / `diamond`菱形 / `all`全图 / `self`自身 |
| range_min / range_max | int | 范围距离 |
| target | enum | `enemy` / `ally` / `self` |
| cooldown | int | 冷却回合数（主动技2—4） |
| rage_cost | int | 怒气消耗，绝技统一100 |
| effects | string | **原子效果序列**，分号分隔，按顺序执行 |
| desc | string | 技能文案（显示用） |

### 原子效果词表（effects字段的合法词汇）

| 类别 | 效果 |
|---|---|
| 伤害/治疗 | `phys_dmg(倍率)` `mgc_dmg(倍率)` `heal(倍率)` `bleed(回合)` `burn(回合)` |
| 位移 | `pull(格)` `pull_to_front` `push(格)` `swap_position` `teleport(格)` |
| 控制 | `stun(回合)` `sleep(回合)` `sleep_chance(概率,回合)` `paralyze(回合)` `bind(回合)` |
| 增减益 | `buff(属性,值,回合)` `debuff_mgc(值,回合)` `def_up(值,回合)` `dodge_up(值,回合)` `block_up(值,回合)` `armor_break(值,回合)` `move_mod(值,回合)` `random_buff(A\|B)` |
| 机制 | `steal_buff(数)` `dispel(数)` `summon(物件)` `aura(属性,值,半径)` `guard(回合)` `counter(回合)` `extra_action(数)` `av_mod(值)` `refresh_on_kill(1)` `execute_below(比例)` `sure_hit` `random_target(数)` `friendly_fire(概率)` `hit_rate(值)` `rage(值)` |
| 修正 | `bonus_vs_elite(倍率)` `bonus_vs_high_def(值)` `bonus_vs_cavalry(值)` `bonus_by_self_lost_hp(系数)` `target_rule(lowest_hp)` |
| 连击 | 任何伤害效果后可跟 `xN` 表示N段（如 `phys_dmg(0.9)x4`） |

**新增技能 = 在表中加一行拼效果，程序零介入**（对应策划文档6.7效果原子化）。未收录的效果需先在 EffectSystem 中实现并更新本表。

## 四、Godot 接入约定

1. 启动时由 `DataLoader`（AutoLoad）解析三张CSV → 生成 `UnitData` / `TerrainData` / `SkillData` 自定义Resource并缓存，全项目通过ID查询
2. 战斗内所有数值读取必须走Resource，禁止硬编码
3. `effects` 字段由 EffectSystem 按词表逐个解析执行；解析失败要报错并指出技能ID与效果名
4. 调参流程：策划改CSV → 重进游戏即生效，不改代码、不重导出

## 五、数据校验规则（建议做成编辑器工具脚本）

- 所有 `skill_signature` 必须能在技能表中找到
- 所有 `bonds` 中的ID必须是已实装或登记过的预留名
- `quality`、`class`、`type`、`range_shape`、`target` 必须在枚举范围内
- 同名ID不得重复；数值不得为负（move_cost 99 除外）
